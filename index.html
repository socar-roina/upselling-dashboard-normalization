수락 경로: 직접 vs 간접

PF0 수락을 추천 화면에서 바로 수락(직접)과 추천 화면을 닫고 결제 페이지에서 PF0를 고른 경우(간접)로 구분
비중(92.6% 대 7.4%)은 정규화 내내 안정적 (아래 건수는 정규화 초기 5/21~6/13 기준)
직접 (추천 화면에서 결제하기로 PF0 수락) 9,946건 92.6%
간접 (추천 화면 닫고 결제 페이지에서 PF0 선택) 797건 7.4%
직접 = 추천 화면의 결제하기 버튼을 누르고 PF0로 결제한 건
간접 = 추천 화면을 닫았지만 이어진 결제 페이지에서 PF0를 직접 고른 건
대부분(92.6%)이 추천 화면에서 바로 전환.

정의

• 예약: 정규화 후 전체 생성 예약 건수 (profit_socar_reservation, created_at_kst 2026-05-21 14:00 ~ 6/13). 240,634건 = 퍼널 모수.
• 노출 고객 / 노출 횟수: 앱 이벤트 로그에서 page_name='upsell_bottomsheet', event_name='view', PF 넛지(upselling_type=pfNudge 또는 단축 ut=p) 이벤트. 고객 수 = 고유 member_id, 노출 횟수 = (날짜 × 고객) 단위.
• 수락(PF0 전환): 노출 고객이 같은 날 실제로 PF0(pf_type='0') 예약을 생성한 경우. 단순 클릭이 아니라 결제 결과 기준.
• 수락률 = 수락(전환) ÷ 노출 횟수 = 5.8%. 노출 고객 중 그날 실제 예약 완료(152,510건, 83%)만 분모로 보면 7.0%(실제 예약자 기준). 노출 전체 중 17%는 추천만 보고 미예약이라 수락 기회 없음.
• 직접/간접: 수락 건 중 추천 화면 결제하기(cta_button) 클릭 후 PF0 결제면 직접(9,946건, 92.6%), cta 클릭 없이 결제 페이지에서 PF0 선택이면 간접(797건, 7.4%).
• 노출 대상: 추천은 예약 1건당 한 종류만 노출. 이미 PF0거나 시간연장/차종 추천이 먼저 뜬 예약은 PF0 노출에서 제외(전체 예약의 76.4%만 PF0 추천 노출).
• 참고: 전체 예약 중 PF0 선택 비율은 30.2%(이미 PF0인 고객 포함)로, 위 수락률(넛지가 PF70/30을 PF0로 전환한 비율)과 다른 지표.

• 결제와 예약(PF0 전환, 보험매출): socar-data.socar_biz_profit.profit_socar_reservation (pf_type, _rev_pf, created_at_kst). 결제 데이터는 D+1~2 반영이라 누적 수치는 6/13까지로 집계.

-- 일별 노출 / PF0 수락 (정규화 배포 후, 구버전/신버전 form_data 통합)
WITH v AS (
  SELECT DISTINCT DATE(event_at_kst) AS d, member_id
  FROM `socar-data.app_web_log.socar_app_web_log`
  WHERE page_name='upsell_bottomsheet' AND event_name='view'
    AND (REGEXP_EXTRACT(form_data, r'upselling_type[\\:]+([A-Za-z]+)')='pfNudge'
         OR REGEXP_EXTRACT(form_data, r'ut[\\:]+([a-z])')='p')   -- pfNudge (신포맷 ut=p)
    AND event_at_kst >= DATETIME '2026-05-21 14:00:00'
    AND event_at_kst <  DATETIME '2026-06-14 00:00:00'           -- 결제 안정 구간 6/13까지
    AND member_id IS NOT NULL
),
r AS (
  SELECT DISTINCT DATE(created_at_kst) AS d, member_id
  FROM `socar-data.socar_biz_profit.profit_socar_reservation`
  WHERE pf_type='0'
    AND created_at_kst >= DATETIME '2026-05-21 14:00:00'
    AND created_at_kst <  DATETIME '2026-06-14 00:00:00'
)
SELECT v.d,
  COUNT(DISTINCT v.member_id) AS exposed_uu,
  COUNT(DISTINCT IF(r.member_id IS NOT NULL, v.member_id, NULL)) AS accepted_pf0_uu
FROM v LEFT JOIN r ON v.d=r.d AND v.member_id=r.member_id
GROUP BY 1 ORDER BY 1;
